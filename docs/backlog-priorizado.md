# Backlog priorizado por impacto

Escala de impacto: Alta / Media / Baja.

## P0 — Impacto alto inmediato
1. **Optimizar checkout y contacto (conversión)** — Impacto: **Alta**
   - Reducir fricción en formularios.
   - CTA claros y seguimiento de eventos clave.
2. **Monitoreo de errores y hardening de rutas críticas (estabilidad)** — Impacto: **Alta**
   - Alertas por errores en producción.
   - Revisar rutas de compra y autenticación.

## P1 — Impacto alto a mediano plazo
3. **SEO local para páginas de servicio (SEO local)** — Impacto: **Alta**
   - Metadata local por página.
   - Datos estructurados LocalBusiness.
4. **Optimización de Core Web Vitals en Home y Tienda (velocidad)** — Impacto: **Alta**
   - Optimizar imágenes y scripts no críticos.
   - Mejorar LCP/CLS/INP.

## P2 — Impacto medio
5. **Mejorar discoverability en contenidos de marca (SEO local)** — Impacto: **Media**
6. **Suite de pruebas end-to-end para flujos críticos (estabilidad)** — Impacto: **Media**

## Métrica de priorización
Para cada iniciativa se recomienda puntuar:
- Impacto en ingresos o leads (40%)
- Reducción de riesgo operativo (30%)
- Esfuerzo estimado (20%)
- Tiempo a valor (10%)
